# Предиктивный контроль качества стекла по химическому составу

Автоматическая классификация 7 типов стекла по химическому составу для систем контроля качества на стекольном производстве.

## Результаты

**Лучшая модель:** Random Forest
- Accuracy: 80.5%
- Kappa: 0.73
- Lift на редких классах (G3): 13.0

## Ключевая фишка: Profit Curve

Большинство аналитиков оценивают модели по Accuracy/F1, Однако можно пойти дальше — построить **Profit Curve** с учётом стоимости ошибок:
- True Positive: +10 у.е.
- False Positive: −2 у.е.
- False Negative: −5 у.е.

**у.е.** - условные единицы, которые вы получаете или теряете в том или ином случае.

Это позволяет определить **оптимальный порог классификации**, максимизирующий прибыль, а не просто точность.

## Как запустить

1. Скачайте датасет: [UCI Glass на Kaggle](https://www.kaggle.com/datasets/uciml/glass)
2. Поместите `glass.csv` в папку `data/`
3. Откройте `analysis.Rmd` в RStudio
4. Нажмите **Knit** (Ctrl+Shift+K)
5. Получите обновлённый `analysis.html` со всеми графиками

Все метрики в отчёте вычисляются автоматически — если переобучить модели, цифры обновятся сами.
Так как обучение моделей - это процесс трудоёмкий и он может занять **10 минут и более**.

## Примеры визуализаций

### Profit Curve для класса G1 (32% выборки)
![Profit Curve G1](./output/profit_curve_g1.png)
**Что это за график?**
Простыми словами: Этот график показывает, сколько денег заработает компания, если будет использовать нашу модель для сортировки стёкол.
**Как это работает?**
Представьте, что у вас есть 100 образцов стекла, и вы хотите найти среди них стекло типа G1 (стекло для зданий). Модель для каждого образца вычисляет вероятность: "Это стекло G1 с вероятностью 85%".
**По горизонтальной оси (X):** Какой процент образцов мы проверяем — от 0% до 100%.
**По вертикальной оси (Y):** Сколько денег мы заработали (или потеряли) на этом.
**Ключевые моменты:**
- **Красная линия выше синей** → Random Forest приносит больше прибыли, чем нейросеть
- **Пик графика на уровне 50-55%** → Максимальная прибыль достигается, когда мы проверяем только половину образцов с наивысшей вероятностью
- **После пика прибыль падает** → Если начинать проверять образцы с низкой вероятностью, мы тратим больше на ложные срабатывания, чем зарабатываем
  
**Вывод**
Не нужно проверять все 100 образцов! Достаточно отобрать только топ-50% с наивысшей вероятностью — это принесёт максимальную прибыль (~130 у.е. для RF). Random Forest зарабатывает на 13% больше, чем нейросеть.

### Lift Curve для редкого класса G3 (8% выборки)
![Lift Curve G3](./output/lift_curve_g3.png)
**Что это за график?**
Простыми словами: Этот график показывает, во сколько раз наша модель эффективнее случайного угадывания при поиске редкого типа стекла.
**Как это работает?**
Класс G3 (стекло для автомобилей) встречается очень редко — всего в 8% случаев (17 из 214 образцов). Если выбирать образцы наугад, то из 10 проверенных вы найдёте меньше 1 нужного.
**По горизонтальной оси (X):** Какой процент образцов мы проверяем.
**По вертикальной оси (Y):** Во сколько раз модель лучше случайного выбора (Lift).
**Ключевые моменты:**
**Lift = 13.0 для RF** → На первых 10% выборки модель находит редкое стекло в 12 раз эффективнее, чем случайный выбор!
**Lift = 6.5 для NN** → Нейросеть тоже хороша, но в 2 раза хуже Random Forest
График быстро падает → После проверки топ-20% образцов эффективность снижается

**Вывод**

Если проверить только **10% образцов** с наивысшей вероятностью по мнению Random Forest, вы найдёте 75% всех редких стёкол G3! При случайном выборе вы бы нашли только 10%. Это экономит **90% ресурсов** на проверку.
